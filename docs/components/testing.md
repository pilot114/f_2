## Что тестировать?

Точно нужно тестировать Entity и UseCase. Остальное тестируется опционально.

## Smoke тесты

Выполняются отдельной командой

> composer smoke_test

## Генерация теста с помощью AI-ассистента

Рекомендуется следующий вариант:

- выбираем имя тестируемого класса (через Copy Reference)
- выполняем команду bin/console system:clipClassWithDepends "ИМЯ_КЛАССА"
если команда выполнена не в контейнере, код класса и его нетранзитивные
зависимости попадут в буфер обмена
- вставляете контекст из буфера обмена в AI-ассистента (например,
хорошо подойдёт бесплатный Claude 3.7 Sonnet)
- добавляете промпт, например "create pest test for ИМЯ_КЛАССА class"
- результат копируете в созданный вручную файл теста
- проверяете тест, исправляете недочёты

По мере необходимости, можно ещё автоматизировать, но в целом
такой вариант даёт достаточный контекст для Unit теста и хороший
контроль над процессом.
