# Исправление структуры проекта

## Дата создания
03.10.2025

## Статус
Принятый

---

## Контекст

В процессе развития проекта возникли отступления от изначальной архитектуры, что привело к ухудшению поддержки и расширяемости кода.
Возникла необходимость исправления структуры проекта.

## Решение

- **Директории в src/Domain теперь сгруппированы по подразделениям компании, без исключений**
- **Проведён рефакторинг src/Common (к нему предъявляются повышенные требования качества)**
- **В DI конфиге (config/services.yaml) также проведён рефакторинг**
- **В docs/components/common.md зафикисирована документация по переиспользуемому коду**
