## Вводные

Цель - собрать базу знаний (векторную базу данных) с ключевой информацией о
бизнес-процессах и проектах.

Это даст возможность оптимизировать многие высокоуровневые бизнес-задачи
с помощью LLM, с высокой доступностью и качеством результата.

Ключевые источники данных:
- Oracle
- Gitlab
- Jira
- Confluence

В качестве провайдера данных выступает апи портала (возможно вынесение в отдельный сервис).
В перспективе возможно расширение апи до полноценного MCP сервера, что
даст широкие возможности для активной автоматизации (заведение задач, создание репозиториев и пр.)

## Запрашиваемые данные

Все собираемые данные можно представить в виде структур:
- доменная структура (по логическому разбиению)
- проектная структура (по существующим проектам)

Доменная структура является основной и предпочтительной.  
В идеале, эти структуры должны быть понятным образом связаны друг к другом.

### SW Knowledge Database

    artefacts
        - table
        - procedure
        - Jira task
        - user stories
        ...
    projects:
        project1
            context: [array of context files]
            // types of context?
        ...
    domains:
        domain1 ... // domains (= departments)
        domain2
          subdomain1 ... // subdomains (= services)
          subdomain2
            useCase1 ... // use cases
            useCase2

Прим.: Для каждого уровня может существовать общая сущность
(общий домен, общие таблицы, общая кодовая база), которая
может использоваться для горизонтального взаимодействия.

Каждый артефакт представляется в json, для которого определена операция
преобразования в текст.

Артефакты могут иметь ссылки друг на друга, причем связь может быть разных типов:
композиция, наследование, зависимость и т.д.

Доменная структура должна быть в векторной базе постоянно,
информация по артефактам и проектам может загружаться / обновляться по требованию.

### Артефакты

Все артефакты имеют общий набор полей:

    contentType:
        Пока единственный тип контента - text. Для других типов
        контента в поле content будет даваться ССЫЛКА на файл.
    
    content:
        Содержимое артефакта
    
    type: (table, procedure, jira task, user story)
        Тип артефакта, определяющий набор дополнительных полей
